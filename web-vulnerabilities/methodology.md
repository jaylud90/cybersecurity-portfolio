# Testing Methodology

My web application testing process follows a structured approach:

1. Identify user-controlled input
2. Determine where the input is processed
3. Observe whether processing happens on the client or server
4. Test how input is rendered or used
5. Verify whether proper validation, encoding, and authorization controls exist
6. Document the vulnerability, impact, and recommended fix

## Focus Areas
- Authentication
- Authorization
- Access control
- Input validation
- Output encoding
- API behavior
